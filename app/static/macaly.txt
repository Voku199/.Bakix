#PROMPT 1B — DB extension pro AI
Extend the Bakix database with tables for the AI subsystem.

ADD THESE TABLES:

1. `ai_conversation_history`
   - id (auto-increment primary key)
   - user_id (FK to current user, indexed)
   - role (text — values: "user" or "model")
   - content (text)
   - intent (text, nullable — values: "chat", "generate_page", "modify_page", "confirm", "explain")
   - page_id (text, nullable — reference to generated_pages)
   - created_at (timestamp, default now, indexed DESC)
   - Index on (user_id, created_at DESC)

2. `ai_cache`
   - user_id (FK)
   - query_hash (text — SHA-256 of user_id + prompt)
   - response (json)
   - created_at (timestamp)
   - Composite PK: (user_id, query_hash)
   - 1-hour TTL

3. `generated_pages`
   - page_id (text, primary key — UUID)
   - user_id (FK, indexed)
   - title (text)
   - content_html (text)
   - source_prompt (text)
   - created_at (timestamp)
   - updated_at (timestamp)
   - Index on (user_id, updated_at DESC)

4. `pending_skills`
   - user_id (FK, primary key)
   - step (integer — 1-4)
   - data_json (json)
   - updated_at (timestamp)

5. `user_skills`
   - id (auto-increment PK)
   - user_id (FK, indexed)
   - name (text)
   - description (text)
   - system_prompt (text)
   - created_at (timestamp)
   - Unique constraint on (user_id, name)

EXTEND `bakalari_credentials.settings_json` default value:
{
  "theme": "auto",
  "ai_enabled": true,
  "ai_intent_routing": true,
  "ai_auto_save_pages": true
}

SECURITY: All AI tables must be scoped strictly per user_id. Backend functions must always filter by current user's ID.

This prompt only adds DB schema. No UI, no functions yet.

`callAiProxy(prompt, history, intent, currentPageContent?)`:
// AI calls go through bakix-proxy.onrender.com/ai/ask → Gemini 2.0 Flash

1. Build context string from last 10 turns of history:
   "[Uživatel]: {content}" or "[Bakix AI]: {content}"
   Join with newlines. Max 3000 chars — truncate oldest first.

2. Build system string based on intent:

   intent="chat":
   "Jsi Bakix AI asistent pro české studenty. Pomáháš s Bakaláři,
    školou a učivem. Odpovídej česky, stručně a přátelsky.
    Pokud chce uživatel vygenerovat stránku nebo cvičení,
    připoj na konec odpovědi přesně tento text: ::AWAITING_CONFIRMATION::"

   intent="generate_page":
   "Jsi Bakix AI asistent. Uživatel chce vygenerovat HTML stránku
    (poznámky, shrnutí, cvičení). Vygeneruj kompletní HTML obsah
    (pouze <body> obsah, bez <!DOCTYPE> a <html> tagu).
    Používej <h1>-<h3>, <p>, <ul>, <ol>, <strong>, <em>, <code>, <blockquote>.
    Obsah musí být v češtině, strukturovaný a užitečný pro studenta.
    Vrať POUZE HTML, žádný markdown, žádné vysvětlení okolo."

   intent="modify_page":
   "Jsi Bakix AI asistent. Uživatel chce upravit existující HTML stránku.
    Aktuální obsah stránky: ---\n{currentPageContent}\n---
    Uprav obsah podle požadavku uživatele.
    Vrať POUZE upravené HTML (<body> obsah), žádné vysvětlení."

   intent="explain":
   "Jsi Bakix AI asistent. Uživatel chce vysvětlení pojmu nebo konceptu.
    Odpovídej česky, srozumitelně, s příklady. Max 3 odstavce."

   intent="confirm":
   "Jsi Bakix AI asistent. Uživatel odpovídá ano/ne na tvoji předchozí otázku.
    Podle odpovědi buď proveď akci (generate_page) nebo zdvořile zruš."

3. Server-side fetch POST https://bakix-proxy.onrender.com/ai/ask:
   Headers: Content-Type: application/json
   Body:
   {
     "prompt":  userMessage,
     "context": <context from step 1>,
     "system":  <system from step 2>,
     "model":   "gemini-2.0-flash"
   }

4. Parse response:
   - response.response → AI text
   - Text ends with ::AWAITING_CONFIRMATION:: → awaiting_confirmation: true, strip sentinel from display
   - intent="generate_page" or "modify_page":
     - Response text IS the HTML content
     - Create/update generated_pages row (content_html = response.response)
     - Return { text: "Připravil jsem ti stránku.", intent, page_id, awaiting_confirmation: false }

#PROMPT 11 — Intent classifier + routing
Build the intent classification layer.

INTENT VALUES:
"chat" | "generate_page" | "modify_page" | "confirm" | "explain"

KEYWORD SETS (case-insensitive substring match, Czech stem-style):

GRADE_KEYWORDS = [
  "známk", "průměr", "jednička", "dvojka", "trojka", "čtyřka", "pětka",
  "ohodno", "klasifikac", "rozeber moje", "moje známk", "vážený průměr",
  "matematik", "čeština", "angličt", "němčin", "fyzik", "chemi", "biolog",
  "dějepis", "zeměpis", "informatik"
]

MODIFICATION_KEYWORDS = [
  "změň", "uprav", "přidej", "odeber", "smaž", "doplň", "zkrať", "rozšiř",
  "přepiš", "udělej to ", "místo toho", "spíš", "líp", "víc příkladů",
  "víc vysvětlení", "stručněji"
]

CONFIRMATION_KEYWORDS_YES = [
  "ano", "jo", "jasně", "ok", "okay", "určitě", "souhlasím",
  "jdem na to", "dělej", "jo prosím", "yes"
]

CONFIRMATION_KEYWORDS_NO = [
  "ne", "nechci", "raději ne", "zruš", "nech to být", "nope", "no"
]

EXPLANATION_TRIGGERS = [
  "co je", "co znamená", "vysvětli", "vysvětlíš", "jak funguje", "rozdíl mezi"
]

GENERATE_PAGE_TRIGGERS = [
  "vygeneruj", "vytvoř stránku", "udělej stránku", "připrav mi",
  "naprogramuj", "vytvoř procvičování", "udělej kvíz",
  "připrav cvičení", "shrň mi", "připrav poznámky"
]

CLASSIFIER FUNCTION `classifyIntent(userMessage, lastAssistantMessage, currentPageId, uiMode)`:

Returns: { intent, confirmSubtype? }

Priority order (first match wins):

1. uiMode === "generate_page" → return { intent: "generate_page" }

2. lastAssistantMessage contains "::AWAITING_CONFIRMATION::":
   - YES keywords match → return { intent: "confirm", confirmSubtype: "yes" }
   - NO keywords match → return { intent: "confirm", confirmSubtype: "no" }
   - No match → fall through

3. currentPageId present AND MODIFICATION_KEYWORDS match → return { intent: "modify_page" }

4. GENERATE_PAGE_TRIGGERS match → return { intent: "generate_page" }

5. EXPLANATION_TRIGGERS match → return { intent: "explain" }

6. GRADE_KEYWORD match AND GENERATE_PAGE_TRIGGER match → return { intent: "generate_page" }

7. GRADE_KEYWORD match alone → return { intent: "chat" }

8. Default → return { intent: "chat" }

All matching: lowercase userMessage first, use .includes() for substring.

SENTINEL SYSTEM:
- AI appends "::AWAITING_CONFIRMATION::" to response when asking yes/no
- Frontend strips sentinel before displaying
- Sentinel in last assistant message activates confirm intent on next turn

INTEGRATION WITH PROMPT 9:
- Replace placeholder intent detection in sendAiMessage() with classifyIntent()
- Pass returned intent into callAiProxy()
- Store intent on ai_conversation_history row

AI PROXY CONTRACT:
All AI calls: POST https://bakix-proxy.onrender.com/ai/ask
Request: { prompt, context?, system?, model? }
Response: { response, model, prompt_tokens, response_tokens, timestamp }
Intent → system prompt mapping:
  "chat" → conversational Czech assistant
  "generate_page" → HTML generation (response IS the HTML)
  "modify_page" → HTML modification with existing content injected
  "explain" → explanation assistant
  "confirm" → confirmation handler
Never call /ai/ask from frontend directly — always server-side.

DEBUG MODE:

- Toggle via /?debug=intent URL param

DEBUG PAGE `/debug/intent` (auth required, hidden from nav):
